## Modular Architecture Principles

Our internal modules are designed to be:

- Stand-alone
- API-driven
- Boundary-focused
- Secure by default
- Future NuGet-package ready
- Easy to register and host in ASP.NET Core applications

Typical module structure:

```text
MinooTrading.<Module>.Abstractions
MinooTrading.<Module>.Domain
MinooTrading.<Module>.Application
MinooTrading.<Module>.Infrastructure
MinooTrading.<Module>.Web
```

Key rules:

- Domain logic stays independent from infrastructure.
- Public contracts are separated from internal implementation.
- Cross-module communication uses stable IDs and explicit contracts.
- Public APIs expose safe DTOs, not internal entities.
- Admin and public surfaces have clear access boundaries.
